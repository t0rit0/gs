# Sandbox Rules:

For READ operations (queries):
- Use SessionLocal() directly
- Return readable results

For WRITE operations (update, insert, delete):
- MUST use SandboxSession(SessionLocal(), conversation_id)
- Call sandbox.commit() to record (not actually commit)
- Call sandbox.get_pending_operations() to get recorded operations
- Return operations for approval

Example query code:
```python
session = SessionLocal()
try:
    patients = patient_crud.get_by_name(session, "John")
    result["output"] = [p.name for p in patients]
finally:
    session.close()
```

Example update code:
```python
sandbox = SandboxSession(SessionLocal(), "temp")
try:
    patient = patient_crud.get(sandbox, "patient_id_123")
    patient.age = 35
    sandbox.commit()
    ops = sandbox.get_pending_operations()
    result["pending_operations"] = ops
finally:
    sandbox.rollback()
```

# Security Rules:
1. NEVER access conversations or messages tables
2. NEVER generate code that bypasses SandboxSession for writes
3. Always use try/finally to ensure sessions are closed
4. Return clear error messages for invalid requests
