# 医疗助手开发路线图

## 项目概述

基于DrHyper诊断引擎构建医疗助手系统，采用周迭代开发模式。

---

## 第一阶段：核心MVP开发（第1-6周）

### 第1周：冷启动对话 + 图像分析 + 初始前端界面

#### 🎯 本周目标
完成可演示的对话系统雏形，支持新患者场景（冷启动）、图像分析诊断和初始前端界面

#### 📋 任务清单
1. **项目基础设施搭建**
   - [ ] 创建项目目录结构
     - api/（FastAPI）、core/、models/、utils/、config/、tests/、frontend/
   - [ ] 配置开发环境（虚拟环境、依赖管理）

2. **DrHyper集成（简化版）**
   - [ ] 集成DrHyper核心代码到项目
   - [ ] 创建DrHyperWrapper封装类
     - init_conversation(patient_info): 初始化对话
     - chat(user_message, image_refs): 处理用户消息
   - [ ] 实现内存版对话状态管理
   - [ ] 实现基础API端点
     - POST /api/conversation/init
     - POST /api/conversation/chat
     - GET /api/conversation/{id}

3. **图像分析集成**
   - [ ] 实现ImageAnalyzer接口
   - [ ] 选择并集成多模态LLM（GPT-4o / Claude 3.5 / Qwen-VL）
   - [ ] 实现临时图像存储（本地文件系统）
   - [ ] 在DrHyper中集成图像分析流程
     - 检测图像 → 生成查询 → 调用分析 → 更新图结构

4. **初始前端界面（简化版）**
   - [ ] 创建前端项目（React/Vue基础框架）
   - [ ] 实现基础对话界面
     - 消息列表展示（用户消息 + AI回复）
     - 文本输入框
     - 发送按钮
   - [ ] 实现图像上传功能
     - 文件选择按钮
     - 图像预览
   - [ ] 对接后端API
     - 调用init接口初始化对话
     - 调用chat接口发送消息
     - 展示AI回复

### 第4周：数据更新沙盒 + 审批流程

#### 🎯 本周目标
实现AI生成的数据更新建议和医生审批流程

#### 📋 任务清单
1. **更新沙盒机制**
   - [ ] 设计pending_updates表结构
   - [ ] 实现DrHyper实体提取
     - 从诊断结论中提取结构化数据
     - 生成更新建议
   - [ ] 实现沙盒CRUD服务
   - [ ] 单元测试

2. **审批流程实现**
   - [ ] 审批API端点
     - GET /api/updates/pending（获取待审批列表）
     - POST /api/updates/{id}/approve（批准）
     - POST /api/updates/{id}/reject（拒绝）
     - POST /api/updates/{id}/modify（修改后批准）
   - [ ] 权限控制（仅医生可审批）
   - [ ] 审批日志记录

3. **完整流程测试**
   - [ ] 诊断 → 生成建议 → 审批 → 更新数据库
   - [ ] 沙盒机制安全性验证

4. **诊断报告生成**
   - [ ] 设计报告数据结构
     - 参考requirements中的报告格式（类别、建议、依据、干预步骤）
     - 定义diagnostic_reports表结构
   - [ ] 实现报告生成器
     - 从对话中提取诊断信息
     - 提取诊断分类（主要诊断、次要诊断）
     - 生成诊断建议（药物治疗、生活方式、随访计划）
     - 整理依据和证据（症状、指标、影像、指南）
     - 制定干预步骤
   - [ ] 报告状态管理
     - DRAFT（AI生成）→ REVIEWING（审核中）→ APPROVED（已批准）
   - [ ] 单元测试

### 第4.5周：诊断报告知识库与检索增强

#### 🎯 本周目标
构建诊断报告知识库，实现历史病例检索增强诊疗

#### 📋 任务清单
1. **报告知识库基础功能**
   - [ ] 实现报告CRUD服务
   - [ ] 报告审核API
     - GET /api/reports/pending（待审核列表）
     - GET /api/reports/{report_id}（报告详情）
     - POST /api/reports/{report_id}/approve（批准）
     - POST /api/reports/{report_id}/reject（拒绝）
     - POST /api/reports/{report_id}/modify（修改后批准）
   - [ ] 报告查询接口
     - 按患者查询历史报告
     - 按诊断分类查询

2. **相似病例检索**
   - [ ] 实现相似度计算算法
     - 症状相似度（Jaccard相似度）
     - 指标相似度（欧氏距离）
     - 人口统计相似度
     - 综合相似度加权
   - [ ] 相似病例检索API
     - POST /api/reports/similar
     - 支持top_k参数
   - [ ] 向量索引优化（可选，后期优化）
     - 使用pgvector或类似工具加速检索

3. **与DrHyper集成**
   - [ ] 诊疗前：历史参考注入
     - 在init_conversation时检索相似病例
     - 将历史病例作为上下文注入DrHyper
     - 优化提示词格式
   - [ ] 诊疗后：报告自动生成
     - 对话完成后自动触发报告生成
     - 从DrHyper提取诊断信息
     - 保存草稿到知识库
   - [ ] 测试完整流程
     - 新患者 → 检索历史 → 增强诊断 → 生成报告 → 审核批准 → 入库

4. **前端报告管理界面（简化版）**
   - [ ] 待审核报告列表
   - [ ] 报告详情展示
   - [ ] 批准/拒绝/修改操作
   - [ ] 相似病例展示（侧边栏）
